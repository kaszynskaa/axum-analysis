Workspace Crates

A Cargo workspace (Cargo.toml at the root) containing four published crates in a strict dependency hierarchy:

axum-macros ← proc-macros only, no axum deps
      ↑
axum-core   ← traits + Body, no axum dep
      ↑
axum       ← router, handlers, extractors, serve
      ↑
axum-extra  ← optional add-ons, depends on axum
CrateVersionRole
axum/0.8.9Main framework. Router, all built-in extractors, middleware helpers, response types, WebSocket support, serve().
axum-core/0.5.6Minimal foundation for library authors. Defines FromRequest, FromRequestParts, IntoResponse, Body. Versioned separately for ecosystem stability.
axum-extra/0.12.6Optional batteries, all feature-gated. Typed cookies, typed HTTP headers, TypedPath, Protobuf, JSON Lines.
axum-macros/0.5.1Proc-macro crate. #[debug_handler], #[derive(FromRequest)], #[derive(TypedPath)]. No dependency on axum itself.

Public Entry Points

axum::serve() — axum/src/serve/mod.rs:103

pub fn serve<L, M, S, B>(listener: L, make_service: M) -> Serve<…>

Starts the HTTP server. Accepts a tokio::net::TcpListener and a Router<()>. Drives hyper_util::server::conn::auto::Builder (HTTP/1 + HTTP/2) and spawns one Tokio task per accepted connection.

Router<S> — axum/src/routing/mod.rs:86

pub struct Router<S = ()>

The primary application-builder type. S is the missing state — only Router<()> (state provided via .with_state(s)) can be passed to serve(). Key methods: .route(), .nest(), .merge(), .layer(), .route_layer(), .fallback(), .with_state().

Method helpers — axum/src/routing/method_routing.rs

pub use method_routing::{get, post, put, delete, patch, head, options, trace, any, on};

Each returns a MethodRouter. The argument is any async function whose parameters implement FromRequest/FromRequestParts and whose return type implements IntoResponse.

Handler trait — axum/src/handler/mod.rs:100

pub trait Handler<T, S>: Clone + Send + Sized + 'static {
    fn call(self, req: Request, state: S) -> Self::Future;
}

Blanket-implemented for all async functions of arity 0–16.

FromRequest / FromRequestParts — axum-core/src/extract/mod.rs

Core extractor traits. Implement FromRequestParts for extractors that do not consume the body; implement FromRequest for extractors that may consume the body (must be the last handler argument).

IntoResponse — axum-core/src/response/mod.rs

pub trait IntoResponse {
    fn into_response(self) -> Response;
}

Implemented for tuples, Json<T>, Html<T>, String, Bytes, StatusCode, Result<T, E>, and more.
